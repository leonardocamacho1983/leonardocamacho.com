# Indexing: Google + AI

This project uses a simple indexing baseline for both traditional search engines and AI crawlers.

## Public indexing surfaces

- `robots.txt`: `/public/robots.txt`
- `sitemap.xml`: generated by `/src/pages/sitemap.xml.ts`
- `llms.txt`: `/public/llms.txt`

## Current indexing policy

- Locale content pages are indexable:
  - `/:locale`
  - `/:locale/about`
  - `/:locale/writing`
  - `/:locale/writing/:slug`
  - `/:locale/privacy`
- Launch welcome pages are indexable:
  - `/welcome`, `/pt-br/welcome`, `/pt/welcome`, `/fr/welcome`
- Confirmed funnel pages remain non-indexed by meta robots default (`FlowLayout` default is `noindex`).
- Internal/non-content surfaces are blocked in `robots.txt`:
  - `/studio`
  - `/api/*`

## Google Search Console setup (DNS TXT)

1. Open Google Search Console and add a new **Domain property**.
2. Copy the TXT value Google provides.
3. In your DNS provider, add a TXT record at host `@` (or the root domain) with that value.
4. Wait for DNS propagation, then click **Verify** in Search Console.
5. After verification:
   - Submit sitemap: `https://www.leonardocamacho.com/sitemap.xml`
   - Request indexing for key URLs:
     - one home locale URL
     - one writing archive URL
     - one post URL
     - one welcome URL
   - Review Coverage, Page Indexing, and Core Web Vitals weekly.

## Maintenance rules

- If domain changes, update:
  - `PUBLIC_SITE_URL`
  - `Sitemap:` line in `robots.txt`
  - canonical host in `llms.txt`
- If new indexable routes are added, include them in `sitemap.xml.ts`.
- If new funnel/internal routes are added, decide whether they should be:
  - `index,follow` (content routes), or
  - `noindex` and/or blocked in `robots.txt` (internal workflow routes).

## Search Console tracking workflow

Use this workflow to track query movement and indexing movement systematically from Google Search Console exports.

### Required export (Performance)

From Search Console `Performance` report:
- export as CSV with dimensions:
  - `Query`
  - `Page`
- metrics:
  - `Clicks`
  - `Impressions`
  - `CTR`
  - `Position`

### Optional export (Indexing)

From Search Console `Page indexing`:
- export as CSV containing:
  - `Reason`
  - `Pages`

### Run tracking

```bash
npm run seo:track -- \
  --csv /absolute/path/to/search-console-performance.csv \
  --indexing-csv /absolute/path/to/search-console-indexing.csv \
  --property leonardocamacho.com \
  --window last-28-days
```

### Outputs

Artifacts are written to:
- `/tmp/seo-tracking/<timestamp>-seo-tracking/`

Key files:
- `summary.txt`
- `performance.snapshot.json`
- `performance.movement.json`
- `indexing.snapshot.json` (if provided)
- `indexing.movement.json` (if provided)

Rolling history is maintained at:
- `/tmp/seo-tracking/history.json`
